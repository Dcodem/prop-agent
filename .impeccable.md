## Design Context

### Users
Property managers and landlords managing multiple rental properties. They spend their days firefighting — fielding calls, dispatching contractors, chasing lease renewals, keeping landlord clients happy. They need an interface that helps them triage fast, not one that makes them think. Mobile and desktop, often switching between the two mid-task.

### Brand Personality
**Professional. Competent. Smart.**

The interface should feel like a sharp operations tool — not a toy, not a bloated enterprise app. It earns trust through clarity and restraint, then rewards attention with thoughtful details.

### Aesthetic Direction
**Practical luxury** — Wealthsimple's elegance applied to property operations. Clean surfaces, generous whitespace, warm neutrals with purposeful color. The kind of interface that feels expensive without being showy. Think: a well-organized desk in a nice office, not a flashy dashboard.

**References**: Wealthsimple (ease of use, elegance, warm minimalism)
**Anti-references**: Generic SaaS dashboards with gratuitous gradients, dark-mode-first "developer tools" aesthetic, rainbow status badges

### Design Principles
1. **Color earns its place** — Every color communicates meaning (status, urgency, category). Decorative color is noise. The warm stone palette is the canvas; semantic colors are the signal.
2. **Calm confidence** — The interface should feel like it has everything under control. No visual urgency unless something actually needs attention. Quiet when things are healthy, assertive when they're not.
3. **Practical before pretty** — Every design choice should make the property manager faster. If a visual element doesn't help them triage, act, or understand, it doesn't belong.
4. **Warm, not cold** — Tinted neutrals (warm stone), Plus Jakarta Sans, generous spacing. The interface should feel human and approachable, never clinical or sterile.
5. **Progressive disclosure** — Show the essential first, reveal depth on interaction. Property managers are scanning, not studying.

### Color System
- **Foundation**: Warm stone palette (stone-50 through stone-900) — the dominant visual tone
- **Accent**: Blue (#2563EB) — interactive elements, CTAs, active navigation
- **Semantic**: Red (error/critical), Orange (caution/attention), Amber (warning/moderate), Green (success/healthy), Purple (people/AI features)
- **Hierarchy**: Color = needs attention or action. Neutral = healthy or informational. This is the core rule.

### Typography
- **Primary**: Plus Jakarta Sans — geometric, warm, modern. Used for everything.
- **Icons**: Material Symbols Outlined — variable weight/fill for active states
